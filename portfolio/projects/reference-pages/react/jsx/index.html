JSX

JSX is a syntax extension for JavaScript, designed to by used with React. Web browsers won't understand JSX code, rather a JSX compiler would be needed to translate JSX into regular JavaScript.

JSX elements

A JSX element looks exactly like a HTML element:

<h1>Hello World</h1>

JSX are treated as JavaScript expressions, so they can go anywhere that JS experssions can go. 

const familyMembers = {
    father: <p>Peter</p>,
    mother: <p>Lois</p>,
    ...
}

Nested JSX

You can nest JSX elements inside other JSX elements. If a JSX expression takes up more that one line, wrap it in parentheses.

const div = (
    <div>
        <div>
            <p>Hello World</p>
        </div>
    </div>
);

JSX Outer Elements

A JSX expression must have exactly one outermost element. This is fine:

const div = (
    <div>
        <div>
            <p>Hello World</p>
        </div>
    </div>
);

But this is not fine:

const div = (
    <div>
        <div>
            <p>Hello World</p>
        </div>
    </div>
    <div>
        <p>Howdy</p>
    </div>
);

Rendering JSX

import React from 'react';
import { createRoot } from 'react-dom/client';

const container = document.getElementById('app');
const root = createRoot(container);
root.render(<h1>Hello world</h1>);

createRoot lets you create a root to display React components inside a browser DOM node. It tells you where you want to place the content that is being rendered. The render() method only updates DOM elements that have changed. If you render the exact same thing twice in a row, nothing will happen.
